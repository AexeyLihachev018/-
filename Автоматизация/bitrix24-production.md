# Настройка производственного процесса в Битрикс24

> Компания: гидравлическое оборудование (~40 чел.)
> Продукты: гидроцилиндры, гидростанции, испытательные стенды, гидрошланги
> CRM: Битрикс24 (продажи и сервис настроены, производство — не настроено)

---

## ЧАСТЬ 1: СТРУКТУРА СМАРТ-ПРОЦЕССА «ПРОИЗВОДСТВЕННЫЙ ЗАКАЗ»

### Что такое Смарт-процесс в Б24?

Смарт-процесс — это настраиваемый объект в Б24 (как карточка сделки, но с полями и стадиями под производство). Он живёт в разделе **CRM → Смарт-процессы**.

Вместо того чтобы тащить заказ через воронку продаж, производственный заказ идёт через **свою воронку с производственными стадиями**.

## ЧАСТЬ 4: АВТОМАТИЗАЦИИ ПО СТАДИЯМ (Роботы и Триггеры Б24)

### Стадия 1 → «Новый заказ»
```
Робот: Поставить задачу инженеру-конструктору
  Название: «Разработать чертежи по заказу #{Номер заказа}»
  Ответственный: [Инженер-конструктор]
  Дедлайн: +3 рабочих дня от даты создания
  Описание: ТЗ из поля «Описание»

Робот: Уведомить в чат (внутренний Б24)
  Кому: [Мастер производства], [Снабженец]
  Текст: «Новый заказ #{Номер} для {Клиент}. Дата готовности: {Плановая дата}»
```

### Стадия 2 → «Проектирование»
```
Триггер: Задача инженера перешла в «Выполнено»
  → Автоматически сменить стадию на «Снабжение»

Робот: Если дедлайн задачи просрочен
  → Уведомить РОПа и директора
  Текст: «Задача по заказу #{Номер} просрочена! Инженер: {Ответственный}»
```

### Стадия 3 → «Снабжение»
```
Робот: Поставить задачу снабженцу
  Название: «Закупить материалы для заказа #{Номер заказа}»
  Дедлайн: за 5 дней до плановой даты производства

Триггер: Поле «Срок поставки материалов» — если дата наступила и стадия не сменилась
  → Уведомление снабженцу + РОПу
```

### Стадия 4 → «Производство»
```
Робот: Поставить задачу мастеру
  Название: «Произвести заказ #{Номер}»
  Дедлайн: Плановая дата готовности - 2 дня (запас на ОТК)

Робот: Уведомить клиента (email / WhatsApp через Wazzup)
  Текст: «Ваш заказ #{Номер} запущен в производство.
          Ожидаемая дата готовности: {Плановая дата}»

Триггер: Каждые 3 дня, пока стадия = «Производство»
  → Напоминание мастеру: «Обновите % готовности по заказу #{Номер}»
```

### Стадия 5 → «ОТК»
```
Робот: Поставить задачу ОТК
  Название: «Проверить заказ #{Номер} на соответствие ТЗ»
  Чек-лист:
    □ Проверить геометрические размеры
    □ Провести испытание давлением
    □ Проверить комплектность
    □ Оформить протокол испытаний
  Дедлайн: +1 рабочий день

Условие: Поле «Результат ОТК» = «Отклонён» или «На доработку»
  Робот: Сменить стадию назад на «Производство»
  Робот: Уведомить мастера с комментарием замечаний ОТК
```

### Стадия 6 → «Готов к отгрузке»
```
Робот: Уведомить менеджера (задача)
  Название: «Согласовать отгрузку заказа #{Номер}»
  Текст: «Заказ #{Номер} прошёл ОТК и готов к отгрузке»

Робот: Уведомить клиента (email + WhatsApp)
  Текст: «Уважаемый {Имя клиента}!
          Ваш заказ готов. Согласуйте с менеджером дату отгрузки:
          {Имя менеджера} {Телефон менеджера}»

Робот: Уведомить бухгалтерию
  Текст: «Подготовить документы на отгрузку: Заказ #{Номер}, Клиент: {Клиент}»
```

### Стадия 7 → «Отгружен»
```
Робот: Обновить связанную сделку в CRM
  Стадия сделки → «Выполнена»
  Поле «Дата закрытия» → сегодня

Робот: Поставить задачу менеджеру
  Название: «Получить обратную связь от клиента — Заказ #{Номер}»
  Дедлайн: +7 дней

Робот: Уведомить клиента
  Текст: «Заказ #{Номер} отгружен!
          Если у вас возникнут вопросы — звоните: {Телефон менеджера}.
          Будем рады видеть вас снова!»
```

### Как создать заказ из сделки (связь Продажи → Производство)

1. В карточке сделки (на стадии «Договор подписан») менеджер нажимает **«Создать → Смарт-процесс»**
2. Автоматически заполняются поля: Клиент, Сумма, связанная сделка
3. Менеджер добавляет: Тип изделия, ТЗ, плановую дату
4. Заказ появляется у мастера производства

## ЧАСТЬ 7: ПОШАГОВАЯ ИНСТРУКЦИЯ — КАК НАСТРОИТЬ В Б24

### Шаг 1: Создать Смарт-процесс
```
CRM → ещё (три точки) → Смарт-процессы → Создать
Название: «Производственный заказ»
Иконка: шестерёнка или завод
```

### Шаг 2: Настроить стадии (воронку)
```
В смарт-процессе → Настройки → Стадии → Добавить:
1. Новый заказ (цвет: синий)
2. Проектирование (цвет: фиолетовый)
3. Снабжение (цвет: жёлтый)
4. Производство (цвет: оранжевый)
5. ОТК (цвет: красный)
6. Готов к отгрузке (цвет: голубой)
7. Отгружен / Закрыт (цвет: зелёный — финальная)
```

### Шаг 3: Добавить поля
```
В смарт-процессе → Настройки → Поля → Добавить поле
(см. таблицу полей в ЧАСТИ 3)
```

### Шаг 4: Настроить автоматизацию
```
В смарт-процессе → Автоматизация → выбрать стадию → Добавить робота
(см. роботы в ЧАСТИ 4)
```

### Шаг 5: Настроить права доступа
```
Настройки → Права доступа:
- Мастер производства: видит и редактирует только «Производство»
- Снабженец: видит только «Снабжение» и «Производство»
- ОТК: видит и редактирует только «ОТК»
- РОП, директор: полный доступ
- Менеджеры: создание + просмотр (без редактирования чужих)
```

### Шаг 6: Добавить на главную страницу
```
Б24 Главная → Редактировать → Добавить виджет
→ «Смарт-процессы» → выбрать «Производственный заказ»
→ Вид: Канбан
```

## ЧАСТЬ 8: ИНТЕГРАЦИЯ С 1С

> Пока производственный учёт ведётся в 1С, данные дублируются вручную.
> Будущая задача: настроить синхронизацию Б24 ↔ 1С через модуль «1С-Битрикс».

| Данные | Откуда | Куда |
|---|---|---|
| Заказы клиентов | Б24 CRM → Смарт-процесс | 1С (вручную пока) |
| Расход материалов | 1С | Б24 (вручную пока) |
| Накладные / счета | 1С | прикрепляются к карточке Б24 (файлом) |
| Статус отгрузки | 1С | → Б24 (можно через API 1С-Битрикс) |

**Рекомендация:** Начать с ручного переноса, настроить дисциплину,
затем автоматизировать через штатный модуль «1С + Битрикс24».

## ЧАСТЬ 9: КЛЮЧЕВЫЕ ВЫГОДЫ ПОСЛЕ НАСТРОЙКИ

| До (как сейчас) | После (Б24 Смарт-процесс) |
|---|---|
| Статус заказа — только у мастера в голове | Все видят статус в реальном времени |
| Клиент сам звонит «когда будет готово?» | Автоуведомления клиенту при смене стадии |
| РОП тратит 30–60 мин/день на статусы | 0 минут — всё обновляется в Б24 |
| Задачи передаются устно | Задачи ставятся автоматически с дедлайном |
| Просрочки заметны постфактум | Уведомления за N дней до просрочки |
| Нет истории заказа | Полная история: кто, что, когда изменил |
| Директор запрашивает отчёт у РОПа | Канбан-дашборд открыт 24/7 |
