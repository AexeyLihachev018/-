# Процесс: Отчёт РК Маркетинг — входящий трафик по источникам

> Таблица: «5.3. Отчет РК маркетинг» (Google Sheets)
> Цель: ежедневный учёт входящих обращений по источникам и типам продукции
> Аудитория: Директор + Руководитель отдела продаж

---

## 1. ЧТО ЗАПУСКАЕТ ПРОЦЕСС? (триггер)

**Основной триггер:** Конец рабочего дня (или начало следующего)

Сотрудник (РОП или маркетолог) собирает данные за прошедший день:
- Смотрит в Яндекс.Директ — сколько переходов / заявок
- Смотрит в Яндекс.Метрику или CRM — SEO-трафик
- Считает входящие письма на почте
- Считает входящие звонки (из телефонии или CRM)
- Подсчитывает обращения старых клиентов из Б24
- Считает заявки через ФОС (форма обратной связи) / Carrotquest
- Прочий трафик — мессенджеры, выставки, сарафан

**Периодичность:** Ежедневно, итог за неделю — пятница или понедельник

## 2. ПОШАГОВЫЕ ДЕЙСТВИЯ

### Шаг 1: Сбор данных по источникам
| Кто | Действие | Откуда данные | Время |
|---|---|---|---|
| РОП / маркетолог | Открывает Яндекс.Директ | Личный кабинет Директа | 5–10 мин |
| РОП / маркетолог | Смотрит SEO-трафик | Яндекс.Метрика / Google Analytics | 5 мин |
| РОП / маркетолог | Считает письма на почте | Email (Яндекс / Gmail) | 5 мин |
| РОП / маркетолог | Считает входящие звонки | Телефония (Мango / Телефон.ру) или Б24 | 5 мин |
| РОП / маркетолог | Считает обращения старых клиентов | Б24 CRM | 5 мин |
| РОП / маркетолог | Считает заявки с ФОС / Carrotquest | Carrotquest кабинет | 5 мин |
| РОП / маркетолог | Прочий трафик | Из головы / блокнот | 3 мин |

**Итого на сбор: 30–40 минут в день**

---

### Шаг 2: Разнесение по типам продукции
Каждое обращение нужно отнести к типу изделия:

| Тип изделия | Обозначение |
|---|---|
| Гидроцилиндр индивидуальный (проект) | ГЦ инд. проект |
| Гидроцилиндр серийный | ГЦ серийка |
| Гидроцилиндр ЕЦЖ (Елец) | ГЦ - ЕЦЖ |
| Гидростанция | ГС |
| Испытательный стенд | Исп. стенды |
| Прочая гидравлика (МС, ФЭС, установки раздачи) | Прочая гидравлика |
| РВД + комплектующие | РВД + компл |
| Мехобработка (токарка) | МО (токарка) |
| Мехобработка (фрезеровка) | МО (фрезеровка) |
| Мехобработка (эрозия) | МО (эрозия) |
| Ремонт (сервисный центр) | Ремонт(СЦ) |
| Пневмоцилиндры | Пневмоцилиндры |

**Кто делает:** РОП вручную смотрит каждое обращение и решает, к какой категории отнести
**Время:** 10–20 минут дополнительно
**Проблема:** Субъективность — одно и то же обращение разные люди могут отнести в разные категории

---

### Шаг 3: Внесение данных в Таблицу 1 (левая)
- Открывает Google Sheets → вкладка текущего месяца
- Находит нужную неделю (Неделя 1–5)
- Вносит цифры по каждому типу изделия × каждому источнику трафика
- Структура: строки = типы изделий / недели, столбцы = источники

**Время:** 10–15 минут

---

### Шаг 4: Внесение данных в Таблицу 2 (правая)
- Та же информация, но в другом разрезе — сводная по источникам для каждого типа продукции
- Формулы должны тянуть из Таблицы 1, но часть данных вносится отдельно вручную
- Проверяет итоговую строку «Итого» и «Общее»

**Время:** 5–10 минут

---

### Шаг 5: Проверка диаграммы
- Диаграмма обновляется автоматически на основании Таблицы 2
- Визуально проверяет, что данные отображаются корректно
- При необходимости — корректирует масштаб или диапазон данных

**Время:** 2–5 минут

---

### Шаг 6: Отправка отчёта (если требуется)
- Делает скриншот или экспортирует в PDF
- Отправляет директору / в общий чат (Telegram / Б24)

**Время:** 3–5 минут

**Итого на весь процесс: ~60–90 минут в день**

| Результат | Формат | Получатель |

**Ключевые выводы из отчёта:**

- На какой тип продукции больший спрос
- Динамика по неделям (растём или падаем)
- Эффективность рекламных кампаний (Директ vs SEO)

## 4. ГДЕ МОЖНО СЛОМАТЬСЯ? (типичные ошибки)

| # | Проблема | Последствие | Частота |
|---|---|---|---|
| 1 | Забыли занести данные за день | Неделя посчитана неверно | Часто |
| 2 | Одно обращение посчитано дважды (звонок + письмо от одного клиента) | Завышенные показатели | Средне |
| 3 | Неправильно отнесли тип изделия | Перекосы в аналитике по продуктам | Часто |
| 4 | РОП заболел / в отпуске | Данные не вносятся несколько дней | Редко, но критично |
| 5 | Изменили структуру таблицы → сломались формулы | Диаграмма не строится | Редко |
| 6 | Данные из Директа расходятся с данными CRM | Непонятно, чему верить | Часто |
| 7 | Заявки из Carrotquest = лиды из Б24 — дубли не отсеяны | Двойной счёт | Часто |
| 8 | Нет единого критерия «что считать обращением» | Разные люди считают по-разному | Системная проблема |

## 5. ПРЕДЛОЖЕНИЕ ПО АВТОМАТИЗАЦИИ

### Цель автоматизации
Полностью убрать ручной ввод данных. Таблица заполняется сама из источников.
РОП тратит **0 минут** вместо 60–90 минут в день.

---

### Вариант А: Быстрый — Telegram Bot + Google Sheets (1–2 недели)

```
Сотрудник (или бот) → Telegram → Apps Script → Google Sheets

Как работает:
1. Бот пишет сотруднику каждый вечер в 17:30:
   «Привет! Введи данные за сегодня:
   1. Количество звонков
   2. Письма на почте
   3. Заявки с сайта (ФОС)
   4. Прочее»

2. Сотрудник отвечает цифрами прямо в Telegram

3. Бот записывает данные в нужную ячейку Google Sheets
   (текущий месяц, текущая неделя, правильная строка)

4. Таблица обновляется автоматически
```

**Что автоматизируется:** Ввод данных, выбор нужной ячейки, итоги
**Что остаётся вручную:** Классификация по типу изделия (субъективно)
**Инструменты:** Google Apps Script + Telegram Bot API
**Стоимость:** 0 руб (всё бесплатно)

---

### Вариант Б: Оптимальный — Автоимпорт из источников (1–2 месяца)

```
[Яндекс.Директ API]  ──┐
[Яндекс.Метрика API] ──┤
[Телефония API]      ──┼──► [Make / Albato] ──► Google Sheets
[Б24 CRM API]        ──┤                         (авто-заполнение)
[Carrotquest API]    ──┘

Каждый день в 00:00 — автоматический сбор данных за вчера
Таблица обновляется сама без участия человека
```

**Что автоматизируется:** Весь сбор данных по источникам
**Что остаётся:** Только проверка и интерпретация
**Инструменты:** Make (Integromat) или Albato + API каждого сервиса
**Стоимость:** Make от 9$/мес, Albato от 990 руб/мес

---

### Вариант В: Максимальный — Дашборд в Б24 или Looker Studio (2–3 месяца)

```
Все данные → Б24 CRM (источники лидов уже там)
                │
                ▼
         Looker Studio (бесплатный Google инструмент)
                │
                ▼
         Живой дашборд для директора:
         - открывает ссылку → видит актуальные данные
         - фильтр по периоду, типу продукции, источнику
         - диаграммы обновляются автоматически
```

**Что автоматизируется:** Всё. Директор видит живые данные в реальном времени.
**Инструменты:** Looker Studio (бывший Google Data Studio) — бесплатно
**Стоимость:** 0 руб (если данные уже в Б24 и Sheets)

---

### Рекомендуемый план внедрения

```
НЕДЕЛЯ 1–2:
  → Telegram Bot для ввода данных вечером
  → Убираем ручной выбор ячейки
  → РОП тратит 5 мин вместо 60–90 мин

МЕСЯЦ 1–2:
  → Подключить Яндекс.Директ API → автоимпорт кликов/заявок
  → Подключить Б24 API → лиды по источникам автоматически
  → Ручной ввод остаётся только для звонков (если нет IP-телефонии)

МЕСЯЦ 2–3:
  → Looker Studio дашборд для директора
  → Еженедельный авторепорт на email
  → РОП полностью освобождён от заполнения таблицы
```
