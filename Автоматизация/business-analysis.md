## 3. КАНДИДАТЫ НА АВТОМАТИЗАЦИЮ (ТОП-5)

### 🔴 Кандидат 1: ОБРАБОТКА ВХОДЯЩИХ ЗАЯВОК И ВЕДЕНИЕ CRM
**Что сейчас:** Заявки с сайта/email/звонков фиксируются вручную в таблицы или блокноты. Нет единой базы. Заявки теряются.
**Что автоматизировать:** CRM-система (AmoCRM / Битрикс24) + автоматический захват лидов с форм сайта, email, мессенджеров → автоназначение менеджера → постановка задачи.
**Повторяемость:** Каждый день, каждая новая заявка
**Экономия:** 1–2 часа/день на отдел продаж + исключение потерь лидов

---

### 🔴 Кандидат 2: ФОРМИРОВАНИЕ И ОТПРАВКА КОММЕРЧЕСКИХ ПРЕДЛОЖЕНИЙ
**Что сейчас:** Менеджер вручную собирает КП в Word/Excel, копирует цены, отправляет PDF.
**Что автоматизировать:** Шаблонизатор КП (CPQ-логика): менеджер выбирает тип продукта + параметры → система автоматически подставляет цены, сроки, генерирует PDF и отправляет на email клиента.
**Повторяемость:** Каждый новый запрос (5–20 КП в месяц)
**Экономия:** 30–90 минут × 15 КП = ~15–20 часов/месяц на менеджера

---

### 🟠 Кандидат 3: FOLLOW-UP И НАПОМИНАНИЯ ПО СДЕЛКАМ
**Что сейчас:** Менеджер держит в голове или записях, кому и когда перезвонить. Часто забывает — клиент уходит к конкурентам.
**Что автоматизировать:** Автоматические напоминания и цепочки писем/сообщений в CRM: если КП отправлено — через 2 дня автоматический follow-up в WhatsApp/email. Если нет ответа — повторное напоминание менеджеру.
**Повторяемость:** Каждая сделка на этапе КП
**Экономия:** Рост конверсии КП в договор на 15–30%

---

### 🟠 Кандидат 4: КОНТРОЛЬ СТАТУСА ПРОИЗВОДСТВА И УВЕДОМЛЕНИЯ КЛИЕНТОВ
**Что сейчас:** Менеджер вручную уточняет статус заказа у мастера/инженера, затем звонит клиенту. Клиенты сами звонят и спрашивают «когда будет готово».
**Что автоматизировать:** Доска задач по производственным заказам (Trello/Notion/1С) + автоматические уведомления клиенту при смене статуса: «Заказ запущен в производство», «Заказ готов к отгрузке», «Документы отправлены».
**Повторяемость:** Каждый заказ, несколько раз
**Экономия:** 30–60 мин/день у менеджеров + снижение входящих звонков от клиентов

---

### 🟡 Кандидат 5: ЗАЯВКИ НА ЗАКУПКУ И КОНТРОЛЬ ПОСТАВОК
**Что сейчас:** Технолог/инженер вручную формирует список нужных материалов, снабженец вручную обзванивает поставщиков, сроки поставки отслеживаются в Excel или в голове.
**Что автоматизировать:** Таблица/система управления закупками: потребность формируется автоматически из производственного плана → снабженец получает готовый список → статусы поставок обновляются в одном месте → оповещение при задержке.
**Повторяемость:** Каждый новый заказ
**Экономия:** 1–2 часа/день снабженца + исключение простоев производства из-за отсутствия материалов

## 4. ПРИОРИТЕТЫ АВТОМАТИЗАЦИИ

| # | Процесс | Сложность | Эффект | Приоритет |
|---|---|---|---|---|
| 1 | CRM + захват лидов | Низкая | Высокий | **ПЕРВЫЙ** |
| 2 | Авто-follow-up по сделкам | Низкая | Высокий | **ВТОРОЙ** |
| 3 | Шаблонизатор КП | Средняя | Высокий | **ТРЕТИЙ** |
| 4 | Статусы производства + уведомления | Средняя | Средний | **ЧЕТВЁРТЫЙ** |
| 5 | Управление закупками | Средняя | Средний | **ПЯТЫЙ** |

---

### Логика приоритетов

**1-й: CRM + захват лидов**
Это фундамент. Без единой базы клиентов всё остальное не работает. Внедряется быстро (1–2 недели), стоит дёшево (AmoCRM от 500 руб/мес), даёт мгновенный результат — перестают теряться заявки.

**2-й: Авто-follow-up**
Настраивается поверх CRM (почти бесплатно). Напрямую влияет на выручку — больше КП превращаются в договоры без дополнительных усилий менеджеров.

**3-й: Шаблонизатор КП**
Сокращает время на рутину у менеджеров и инженеров. Требует один раз настроить шаблоны. Окупается за 1 месяц.

**4-й: Статусы производства**
Снижает хаос внутри компании и количество «звонков-вопросов» от клиентов. Требует дисциплины от производства в обновлении статусов.

**5-й: Управление закупками**
Важно, но чуть сложнее в интеграции с производственным учётом. Внедрять после того, как наведён порядок в продажах.

## 5. АНАЛИЗ ОТЧЁТНОСТИ РОПа — УЗКИЕ МЕСТА И АВТОМАТИЗАЦИЯ

### Текущая структура отчёта (Excel, заполняется РОПом вручную)

| Таблица | Что содержит | Кто заполняет | Периодичность |
|---|---|---|---|
| Фиолетовая | % выполнения плана, выручка, средний чек, номенклатура | РОП | Ежедневно/итог месяца |
| Голубая | Лиды по неделям, брак, конверсия 9,95% | РОП | Еженедельно |
| Рекламные трафики | Яндекс.Директ, SEO, почта, звонки, Carrotquest | РОП | Еженедельно |
| Зелёная | Ежедневные данные по каждому менеджеру (ЗАПОЛНЯТЬ ЕЖЕДНЕВНО) | РОП | Каждый день |
| Оранжевая | Сводная по менеджерам — формируется из зелёной автоматически | Формулы Excel | Автоматически |

---

### Выявленные узкие места

#### Проблема 1: Ошибки #DIV/0! в оранжевой таблице
**Что происходит:** У менеджеров Бабурин И. и Микрюкова А. нет данных по старой базе (0 обращений), формула делит на ноль.
**Риск:** Некорректная статистика → неверные управленческие решения.
**Решение:** Заменить формулы на `=IFERROR(A/B; 0)` или `=IF(B=0; 0; A/B)`.

#### Проблема 2: Ручное заполнение зелёной таблицы ежедневно
**Что происходит:** РОП вручную вносит данные по каждому менеджеру каждый день. При 5 менеджерах × 7 полей = 35 ячеек в день.
**Риск:** Ошибки ввода, задержки, потеря данных при болезни РОПа.
**Решение:** Данные берутся автоматически из Битрикс24 через API или экспорт.

#### Проблема 3: Отчёт размазан по месячным вкладкам
**Что происходит:** Каждый месяц — отдельная вкладка (видно: Август, Сентябрь ... Апрель 2026). Сравнение между месяцами требует ручного переключения и копирования.
**Риск:** Нет сквозной аналитики, тренды не видны.
**Решение:** Единая сводная вкладка с выбором периода через срезы/фильтры.

#### Проблема 4: Рекламные трафики не связаны с лидами в CRM
**Что происходит:** Данные из Яндекс.Директ, SEO, звонков вносятся вручную, без сверки с Б24.
**Риск:** Расхождение данных между CRM и отчётом.
**Решение:** Подключить автоматический импорт: Яндекс.Директ → Б24 (уже настроено) → автоэкспорт в таблицу.

#### Проблема 5: Конверсия 9,95% — нет дрилл-дауна
**Что происходит:** Одна цифра конверсии на весь отдел скрывает реальную картину по менеджерам (диапазон: 5% у Ожегова — 100% у Лихачева).
**Риск:** Директор видит среднюю температуру по больнице, не видит проблемных менеджеров.
**Решение:** Добавить визуализацию конверсии по каждому менеджеру отдельно.

### Рекомендации по единому документу (замена Excel-отчёта)

#### Вариант А: Улучшить текущий Excel (быстро, бесплатно)
1. Исправить ошибки #DIV/0! во всех формулах
2. Добавить сводную вкладку «Год» с автоподтягиванием из месячных вкладок
3. Настроить условное форматирование: красный — план < 80%, жёлтый — 80–99%, зелёный — ≥ 100%
4. Добавить мини-графики (sparklines) по неделям для каждого менеджера
5. Защитить оранжевую таблицу от случайного редактирования

#### Вариант Б: Перенести в Google Sheets + автообновление (оптимально)
1. Google Sheets → общий доступ для директора в реальном времени
2. Подключить **Albato** или **Make (Integromat)**: Битрикс24 → Google Sheets автоматически обновляет зелёную таблицу
3. РОП больше не заполняет данные вручную — только проверяет
4. Директор видит актуальный отчёт в любой момент без запроса у РОПа

#### Вариант В: Дашборд в Битрикс24 (максимальная автоматизация)
1. Все данные уже есть в Б24 — нужно только настроить отчёты
2. В Б24 есть встроенный **CRM Аналитика**: воронка, конверсии, выручка по менеджерам
3. Добавить виджеты на главную страницу Б24 для директора
4. Еженедельный автоотчёт на email директора из Б24
